# MAKS BG Remover — Full-Stack Web App Build Prompt

## Project Overview
Build a production-ready, single-page web application called **MAKS BG Remover** — an AI-powered background removal tool. The app accepts an image from the user, sends it to an n8n webhook (which runs a ClipDrop background-removal workflow), receives the processed image, and lets the user download it in their desired format and size. The design must be stunning, modern, and professional with zero placeholder content.

---

## Tech Stack
- **Frontend:** Vanilla HTML5 + CSS3 + ES2022 JavaScript (no frameworks required; if a framework is used, prefer React or Next.js)
- **Styling:** Custom CSS with CSS variables for theming; no Tailwind required but acceptable
- **HTTP:** Native `fetch` API with `AbortController` for timeouts
- **Image processing:** Browser Canvas API for client-side resize, crop, and format conversion
- **Security:** Input validation, file-type whitelist, size limits, sanitised error messages, no secrets in client code

## Section 2 — Upload Zone

### Layout
Centred card, max-width 960 px, with generous padding and a rounded border (2 px dashed, purple-tinted at 40 % opacity) that glows brighter on hover and drag-over.

### Three Input Methods — all must work simultaneously

#### A. Click-to-Browse
- The entire upload zone must be a `<label for="fileInput">` element
- The `<input type="file" id="fileInput">` is `position: absolute; width: 1px; height: 1px; opacity: 0` — visually hidden but fully accessible and native-click-triggerable
- **Do NOT use `fileInput.click()` in JavaScript** — this is blocked in sandboxed iframes; the label association is the only reliable method
- A styled `"Choose Image"` span inside the label acts as the visual button

#### B. Drag & Drop
- Attach `dragover`, `dragleave`, and `drop` listeners to the label/zone element
- On `dragover`: add `.drag` class (brightens border, lightens background, slight scale)
- On `drop`: extract `event.dataTransfer.files[0]`, validate, then call `processFile(file)`
- On `dragleave`: remove `.drag` class

#### C. Clipboard Paste (Ctrl+V / Cmd+V)
- Attach a global `paste` event listener to `document`
- Extract image from `event.clipboardData.items`
- Filter items where `item.type.startsWith('image/')`
- Call `item.getAsFile()` then pass to `processFile(file)`
- Show a brief "📋 Image pasted!" toast notification (bottom-right, 2 s, then fade out)

### Accepted Formats
`image/jpeg`, `image/png`, `image/webp`, `image/bmp`, `image/gif`, `image/tiff`, `image/avif`

### Validation (client-side, before upload)
- File type must be in the accepted list — show error if not
- File size must not exceed **20 MB** — show error if exceeded
- If file is valid, immediately show the original image preview in a small thumbnail next to the zone with filename + dimensions + file size

## Section 4 — Webhook Integration

### Endpoint
```
POST https://staffcollegemultan.app.n8n.cloud/webhook-test/uploadimage
```

### Critical: Avoid FormData / postMessage Cloning Error
**Do NOT use the `FormData` API.** In sandboxed environments (iframes, certain hosting contexts), `FormData` objects cannot be cloned through `postMessage`, causing a hard runtime error.

Instead, **manually construct the `multipart/form-data` body as an `ArrayBuffer`**:

```javascript
async function buildMultipartBody(file) {
  return new Promise((resolve, reject) => {
    const reader = new FileReader();
    reader.onerror = () => reject(new Error('File read failed'));
    reader.onload = (e) => {
      const boundary = '----MAKSBoundary' + Math.random().toString(36).slice(2);
      const filename = file.name || 'image.png';
      const mimeType = file.type || 'image/png';
      const header = `--${boundary}\r\nContent-Disposition: form-data; name="image"; filename="${filename}"\r\nContent-Type: ${mimeType}\r\n\r\n`;
      const footer = `\r\n--${boundary}--\r\n`;
      const hBytes = new TextEncoder().encode(header);
      const fBytes = new TextEncoder().encode(footer);
      const fileBytes = new Uint8Array(e.target.result);
      const body = new Uint8Array(hBytes.length + fileBytes.length + fBytes.length);
      body.set(hBytes, 0);
      body.set(fileBytes, hBytes.length);
      body.set(fBytes, hBytes.length + fileBytes.length);
      resolve({ body: body.buffer, contentType: `multipart/form-data; boundary=${boundary}` });
    };
    reader.readAsArrayBuffer(file);
  });
}
```

Then send:
```javascript
const { body, contentType } = await buildMultipartBody(file);
const response = await fetch(WEBHOOK_URL, {
  method: 'POST',
  headers: { 'Content-Type': contentType },
  body,
  signal: AbortSignal.timeout(90000) // 90-second timeout
});
```

### Response Handling
The n8n workflow responds via a "Respond to Webhook" node in **Binary** mode, returning the processed PNG directly. Handle all response formats gracefully:

1. **Read the entire response as `ArrayBuffer`** (do not rely on content-type headers alone)
2. **Inspect magic bytes** to detect format regardless of headers:
   - PNG: `bytes[0] === 137 && bytes[1] === 80`
   - JPEG: `bytes[0] === 255 && bytes[1] === 216`
   - WEBP: `bytes[8] === 87 && bytes[9] === 69` (ascii "WE" at offset 8)
3. If bytes are too small (< 50 bytes), throw: *"Response too small — set Respond to Webhook → Binary mode in n8n"*
4. If none of the above match, attempt JSON parse and look for `url`, `image` (base64), or `data` fields
5. Create a blob URL with `URL.createObjectURL(new Blob([buf], { type: detectedMime }))`

### Error Handling
- Network/fetch failure: friendly message + retry button
- HTTP 4xx/5xx: show status code + truncated response body (200 chars)
- Timeout (AbortError): *"Request timed out after 90 s — check if n8n workflow is active"*
- All errors must show in a styled error card (red-tinted) with an option to try again — never expose raw stack traces to the user

## Security Measures

| Concern | Mitigation |
|---|---|
| Malicious file type | Validate both `file.type` MIME and magic bytes before sending |
| Oversized uploads | Enforce 20 MB client-side hard limit |
| XSS via filename | Never inject raw filename into innerHTML; use `textContent` or sanitise |
| Webhook URL exposure | Acceptable for client-side tool; document that the URL should be a production webhook (not test) in prod |
| Response injection | Never `eval()` or inject raw server response into DOM |
| Blob URL leaks | Always call `URL.revokeObjectURL()` after use |
| Timeout/DoS | 90-second `AbortController` timeout on all fetch calls |
| Canvas tainting | Result image is a local blob URL — no cross-origin canvas issues |

## Performance

- No external dependencies or CDN scripts
- CSS animations use `transform` and `opacity` only (GPU-composited, no layout thrashing)
- Particle system capped at 25 elements, removed if `prefers-reduced-motion` is set
- Images displayed via blob URLs (no data URI memory bloat for large files)
- Canvas operations run synchronously only when user clicks download (not on every slider move)
